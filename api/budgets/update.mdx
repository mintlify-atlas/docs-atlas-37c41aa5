---
title: "Update Budget"
api: "PATCH /api/budgets/:id"
description: "Modify an existing budget"
---

## Authentication

This endpoint requires JWT authentication via Bearer token.

## Path Parameters

<ParamField path="id" type="string" required>
  UUID of the budget to update
</ParamField>

## Request Body

All fields are optional. Only provide the fields you want to update.

<ParamField body="amount" type="number">
  New budget limit amount. Must be a positive number.
  
  **Validation**: Must be greater than 0
</ParamField>

<ParamField body="month" type="integer">
  New month for the budget (1-12)
  
  **Validation**: Min: 1, Max: 12
</ParamField>

<ParamField body="year" type="integer">
  New year for the budget
  
  **Validation**: Min: 2024
</ParamField>

<ParamField body="categoryId" type="string">
  New category UUID to associate with this budget
  
  **Validation**: Must be a valid UUID
</ParamField>

## Validation Rules

### Ownership
You can only update budgets you own.

**Error**: `403 Forbidden` - "No tienes permiso"

### Parent Category Constraint
If updating the amount and the category has a parent with a budget, the sum of all sibling budgets cannot exceed the parent's limit.

**Error**: `400 Bad Request` - "El presupuesto supera el límite de la categoría padre. Disponible: $X."

### Child Categories Constraint
If updating the amount and the category has child categories with budgets, the new amount must be greater than or equal to the sum of all child budgets.

**Error**: `400 Bad Request` - "El monto es menor a la suma de las subcategorías ($X). Ajusta las subcategorías primero."

### Not Found
Budget must exist.

**Error**: `404 Not Found` - "Presupuesto no encontrado"

## Important Notes

- When updating the amount, both parent and child constraints are validated
- If you need to reduce a parent budget below child budgets total, adjust child budgets first
- Partial updates are supported - only send the fields you want to change
