---
title: 'Get Transaction'
api: 'GET /api/transactions/:id'
description: 'Retrieve a specific transaction by its ID'
---

## Authentication

This endpoint requires authentication using a Bearer token.

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Path Parameters

<ParamField path="id" type="string" required>
  UUID of the transaction to retrieve
</ParamField>

## Response

Returns a single transaction object with full details.

<ResponseField name="id" type="string">
  Unique identifier for the transaction
</ResponseField>

<ResponseField name="type" type="string">
  Transaction type (INCOME, EXPENSE, or TRANSFER)
</ResponseField>

<ResponseField name="amount" type="number">
  Transaction amount
</ResponseField>

<ResponseField name="currency" type="string">
  Currency code
</ResponseField>

<ResponseField name="description" type="string">
  Transaction description
</ResponseField>

<ResponseField name="date" type="string">
  Transaction date in ISO 8601 format
</ResponseField>

<ResponseField name="userId" type="string">
  ID of the user who owns this transaction
</ResponseField>

<ResponseField name="accountId" type="string">
  ID of the associated account
</ResponseField>

<ResponseField name="categoryId" type="string">
  ID of the associated category (if any)
</ResponseField>

<ResponseField name="account" type="object">
  Full account object
  <expandable title="account fields">
    <ResponseField name="id" type="string">
      Account UUID
    </ResponseField>
    <ResponseField name="name" type="string">
      Account name
    </ResponseField>
    <ResponseField name="type" type="string">
      Account type (WALLET, SAVINGS, etc.)
    </ResponseField>
    <ResponseField name="balance" type="number">
      Current account balance
    </ResponseField>
    <ResponseField name="currency" type="string">
      Account currency
    </ResponseField>
  </expandable>
</ResponseField>

<ResponseField name="category" type="object">
  Full category object (if categoryId exists)
  <expandable title="category fields">
    <ResponseField name="id" type="string">
      Category UUID
    </ResponseField>
    <ResponseField name="name" type="string">
      Category name
    </ResponseField>
    <ResponseField name="type" type="string">
      Category type (INCOME, EXPENSE, BOTH)
    </ResponseField>
    <ResponseField name="icon" type="string">
      Category icon
    </ResponseField>
  </expandable>
</ResponseField>

<ResponseField name="createdAt" type="string">
  Timestamp when the transaction was created
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Timestamp when the transaction was last updated
</ResponseField>

<ResponseField name="deletedAt" type="string">
  Soft delete timestamp (null for active transactions)
</ResponseField>

## Error Responses

<ResponseField name="400" type="error">
  Bad Request - Invalid UUID format
</ResponseField>

<ResponseField name="401" type="error">
  Unauthorized - Missing or invalid authentication token
</ResponseField>

<ResponseField name="404" type="error">
  Not Found - Transaction doesn't exist or doesn't belong to the authenticated user
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET https://api.yourfinanceapp.com/api/transactions/a3c4e8f2-9d7b-4a1c-8e3f-2b5d9a7c1e4f \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const transactionId = 'a3c4e8f2-9d7b-4a1c-8e3f-2b5d9a7c1e4f';

const response = await fetch(`https://api.yourfinanceapp.com/api/transactions/${transactionId}`, {
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

const transaction = await response.json();
```

```python Python
import requests

transaction_id = 'a3c4e8f2-9d7b-4a1c-8e3f-2b5d9a7c1e4f'

response = requests.get(
    f'https://api.yourfinanceapp.com/api/transactions/{transaction_id}',
    headers={'Authorization': 'Bearer YOUR_JWT_TOKEN'}
)

transaction = response.json()
```

</CodeGroup>

## Notes

- The endpoint only returns transactions that belong to the authenticated user
- Soft-deleted transactions (where `deletedAt` is not null) are not accessible through this endpoint
- The UUID must be a valid v4 UUID format, otherwise a 400 error is returned
