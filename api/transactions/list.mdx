---
title: 'List Transactions'
api: 'GET /api/transactions'
description: 'Retrieve a paginated list of transactions with optional filtering'
---

## Authentication

This endpoint requires authentication using a Bearer token.

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Query Parameters

### Pagination

<ParamField query="page" type="number" default="1">
  Page number (minimum: 1)
</ParamField>

<ParamField query="limit" type="number" default="20">
  Number of items per page (minimum: 1, maximum: 100)
</ParamField>

### Filters

<ParamField query="search" type="string">
  Search by description or category name (case-insensitive)
</ParamField>

<ParamField query="type" type="enum">
  Filter by transaction type
  <expandable title="values">
    - `income`
    - `expense`
  </expandable>
</ParamField>

<ParamField query="accountId" type="string">
  Filter by account UUID
</ParamField>

<ParamField query="categoryId" type="string">
  Filter by category UUID
</ParamField>

<ParamField query="currency" type="enum">
  Filter by currency
  <expandable title="values">
    - `ARS` - Argentine Peso
    - `USD` - US Dollar
    - `EUR` - Euro
  </expandable>
</ParamField>

<ParamField query="startDate" type="string">
  Filter transactions from this date (ISO 8601 format)
</ParamField>

<ParamField query="endDate" type="string">
  Filter transactions until this date (ISO 8601 format)
</ParamField>

<ParamField query="month" type="number">
  Filter by month (1-12). Requires `year` parameter.
</ParamField>

<ParamField query="year" type="number">
  Filter by year (minimum: 2000). Can be used with `month` parameter.
</ParamField>

<ParamField query="date" type="string">
  Filter by specific date (ISO 8601 format)
</ParamField>

## Example Requests

<CodeGroup>

```bash Basic List
curl -X GET "https://api.yourfinanceapp.com/api/transactions?page=1&limit=20" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash Filter by Type and Date Range
curl -X GET "https://api.yourfinanceapp.com/api/transactions?type=expense&startDate=2026-03-01&endDate=2026-03-31" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash Search and Filter by Account
curl -X GET "https://api.yourfinanceapp.com/api/transactions?search=comestibles&accountId=uuid-de-la-cuenta" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```bash Filter by Month and Year
curl -X GET "https://api.yourfinanceapp.com/api/transactions?month=3&year=2026" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const params = new URLSearchParams({
  page: '1',
  limit: '20',
  type: 'expense',
  startDate: '2026-03-01',
  endDate: '2026-03-31'
});

const response = await fetch(`https://api.yourfinanceapp.com/api/transactions?${params}`, {
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN'
  }
});

const { data, meta } = await response.json();
```

```python Python
import requests

params = {
    'page': 1,
    'limit': 20,
    'type': 'expense',
    'startDate': '2026-03-01',
    'endDate': '2026-03-31'
}

response = requests.get(
    'https://api.yourfinanceapp.com/api/transactions',
    headers={'Authorization': 'Bearer YOUR_JWT_TOKEN'},
    params=params
)

result = response.json()
data = result['data']
meta = result['meta']
```

## Notes

- Transactions are ordered by date in descending order (newest first)
- Only non-deleted transactions are returned (where `deletedAt` is null)
- The search parameter performs case-insensitive matching on both transaction description and category name
- When using `month` and `year` parameters, they override `startDate` and `endDate` if both are provided
