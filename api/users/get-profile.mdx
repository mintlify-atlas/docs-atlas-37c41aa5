## GET /api/users/me

Retrieve the complete profile of the currently authenticated user, including preferences and settings.

### Authentication

This endpoint requires a valid JWT token in the Authorization header.

```bash
Authorization: Bearer <token>
```

### Response

Returns the user profile object with all properties except the password.

<ResponseField name="id" type="string">
  User UUID
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

<ResponseField name="firstName" type="string">
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string">
  User's last name
</ResponseField>

<ResponseField name="avatarUrl" type="string | null">
  URL to user's avatar image
</ResponseField>

<ResponseField name="phone" type="string | null">
  User's phone number
</ResponseField>

<ResponseField name="authProvider" type="string">
  Authentication provider: LOCAL, GOOGLE, MICROSOFT, or APPLE
</ResponseField>

<ResponseField name="currency" type="string">
  User's preferred currency (default: ARS)
</ResponseField>

<ResponseField name="fiscalStartDay" type="number">
  Day of month when fiscal period starts (1-28)
</ResponseField>

<ResponseField name="timezone" type="string">
  User's timezone (default: America/Argentina/Buenos_Aires)
</ResponseField>

<ResponseField name="language" type="string">
  User's preferred language (default: es)
</ResponseField>

<ResponseField name="role" type="string">
  User role: USER or ADMIN
</ResponseField>

<ResponseField name="isActive" type="boolean">
  Whether the user account is active
</ResponseField>

<ResponseField name="subscription" type="string">
  Subscription tier: FREE or PRO
</ResponseField>

<ResponseField name="emailVerified" type="string | null">
  Timestamp of email verification, or null if not verified
</ResponseField>

<ResponseField name="createdAt" type="string">
  Account creation timestamp
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Last update timestamp
</ResponseField>

### Example Request

```bash cURL
curl -X GET http://localhost:3000/api/users/me \
  -H "Authorization: Bearer <token>"
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/api/users/me', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

const user = await response.json();
```

```python Python
import requests

headers = {'Authorization': f'Bearer {token}'}
response = requests.get(
    'http://localhost:3000/api/users/me',
    headers=headers
)

user = response.json()
```

## Difference from /auth/profile

This endpoint (`/users/me`) returns more detailed information than `/auth/profile`, including:
- Subscription tier
- Email verification status
- Fiscal preferences
- Phone number
- Creation and update timestamps

Use `/auth/profile` for quick token validation and basic user info. Use `/users/me` for complete profile data.
